## 一、产品概述

### 1.1 产品定位

惠农网是国内领先的农产品B2B电子商务平台，致力于连接农业生产者、批发商和采购商，为农产品供应链提供信息撮合、在线交易、行情资讯、农技服务等一站式解决方案。平台以“绿色、健康、可追溯”为核心理念，通过数字化手段提升农产品流通效率，助力乡村振兴战略实施。

### 1.2 目标用户

本平台面向以下核心用户群体：

- **供应商群体**：包括农产品种植户、养殖户、合作社、农业企业等，他们需要在平台上发布供应信息、寻找销售渠道、管理店铺和订单。
- **采购商群体**：包括农产品批发商、超市采购商、餐饮企业、食品加工厂等，他们需要寻找优质货源、了解市场行情、进行批量采购。
- **服务商群体**：包括物流服务商、农业金融机构、农资供应商等，为农产品交易提供配套服务支持。
- **农业从业者**：关注农业资讯、学习农技知识、了解市场动态的广大农业从业者。

### 1.3 核心价值

平台的核心价值主张可以概括为“连接、信任、效率”三个维度：

- **连接价值**：打破传统农产品流通中的信息不对称，建立供应商与采购商之间的直接沟通渠道，降低交易成本，拓宽销售渠道。
- **信任价值**：通过实名认证、交易保障、评价体系、信用背书等多重机制，在农产品交易中建立线上信任，降低交易风险。
- **效率价值**：提供智能搜索、精准匹配、在线交易、订单管理等数字化工具，提升农产品交易的效率和便捷性。

### 1.4 产品架构

平台采用“前台展示+后台管理”的架构模式，整体产品架构包含以下核心模块：

- **用户模块**：支持用户注册、登录、个人中心、账户安全、会员体系等功能。
- **供应模块**：支持供应信息发布、编辑、上下架管理、库存管理等功能。
- **采购模块**：支持采购需求发布、报价管理、询价管理等功能。
- **交易模块**：支持在线下单、支付结算、订单管理、物流跟踪、售后服务等功能。
- **店铺模块**：支持商铺开设、店铺装修、商品管理、运营分析等功能。
- **社区模块**：支持行情资讯、农技学堂、社区问答、直播带货等功能。
- **搜索模块**：支持全局搜索、分类筛选、热门推荐、智能推荐等功能。
- **消息模块**：支持系统通知、订单消息、询价消息、客服消息等功能。

## 二、功能需求规格

### 2.1 用户模块

#### 2.1.1 用户注册与登录

用户注册与登录是平台的基础功能，支持多种注册和登录方式，满足不同用户的操作习惯。

**注册功能**：

- 手机号注册：用户输入手机号码，获取并填写短信验证码，设置登录密码，完成注册流程。系统需对手机号格式进行校验，确保手机号有效且未被注册。
- 第三方账号登录：支持微信、支付宝、QQ等主流第三方平台的账号授权登录，降低用户的注册门槛，提升用户体验。
- 企业用户注册：提供企业实名认证通道，企业用户需提交营业执照、法人信息等资质材料，经平台审核通过后方可开通企业账号，享受更多企业级服务。

**登录功能**：

- 账号密码登录：用户输入已注册的手机号或用户名和密码进行登录，支持“记住登录状态”功能。
- 验证码登录：用户输入手机号码，获取短信验证码进行登录，适合密码遗忘或临时登录场景。
- 第三方登录：与注册功能中的第三方平台打通，实现一键登录。
- 登录安全保护：连续登录失败5次后，需等待30分钟方可再次尝试；支持登录设备管理，用户可查看和管理已登录设备。

#### 2.1.2 个人中心

个人中心是用户管理自身信息和资产的核心入口，采用卡片式布局，信息层次清晰。

**账户信息管理**：

- 基本信息：展示头像、昵称、手机号、邮箱、实名认证状态、会员等级等信息。
- 实名认证：支持个人实名认证和企业实名认证，个人需提交身份证信息，企业需提交营业执照等资质材料。认证状态分为“未认证”、“认证中”、“已认证”、“认证失败”四种状态。
- 账户安全：提供密码修改、手机号绑定/更换、邮箱绑定、支付密码设置等功能入口。
- 隐私设置：用户可设置联系方式展示范围、消息推送偏好等隐私选项。

**资产中心**：

- 钱包余额：展示账户可用余额、冻结金额、待结算金额等信息，支持充值、提现操作。
- 交易流水：记录账户的所有收支明细，包含时间、金额、类型、关联订单号等信息，支持按时间范围筛选和导出。
- 优惠券：展示用户持有的优惠券数量和使用状态，支持查看优惠券详情和使用规则。
- 积分账户：展示用户积分余额和积分变动记录，积分可用于兑换礼品或抵扣现金。

**我的订单**：

- 订单分类：分为采购订单、供应订单、商城订单三大类，每个大类下细分更多状态。
- 订单筛选：支持按订单状态（全部、待付款、待发货、待收货、已完成、已取消）、时间范围进行筛选。
- 订单搜索：支持通过订单编号、商品名称、供应商名称等关键词搜索订单。
- 订单详情入口：点击任一订单卡片进入订单详情页。

**我的收藏**：

- 收藏分类：分为商品收藏、店铺收藏、行情收藏、资讯收藏四个分类。
- 批量管理：支持多选收藏内容进行批量删除或移入文件夹。
- 文件夹管理：用户可创建自定义收藏文件夹，对收藏内容进行分类管理。

#### 2.1.3 会员体系

平台建立多层级会员体系，为不同等级会员提供差异化服务和权益。

**会员等级**：

- 注册会员：完成注册即成为注册会员，享受基础浏览和交易功能。
- 认证会员：完成实名认证的会员，解锁发布供求信息、使用询价功能等权限。
- 优选会员：符合平台优选标准的认证会员，获得“惠农优选”标识，提升曝光和信任度。
- 战略会员：平台重点合作企业客户，享受专属服务通道、一对一运营支持等高级权益。

**会员权益**：

- 信息发布权益：不同等级会员每日可发布的信息数量不同，认证会员以上可享受无限制发布。
- 展示排名权益：优选会员和战略会员的信息在搜索结果和分类列表中享有优先展示权。
- 服务费优惠：发布信息、会员续费等服务可享受阶梯式价格优惠。
- 专属客服：战略会员配备专属客服经理，提供7×24小时服务支持。

### 2.2 供应模块

#### 2.2.1 供应信息列表页

供应信息列表页是平台的核心页面之一，承载着商品展示和交易引导的核心功能。

**页面布局**：

- **顶部搜索区**：位于页面最上方，包含关键词搜索框、分类选择器、价格区间输入框、地区选择器、筛选按钮、重置按钮等元素。搜索框支持智能联想和历史搜索记录展示。
- **筛选侧边栏**：页面左侧固定显示分类筛选面板，展示一级分类和二级分类树形结构，支持多选；右侧设置属性筛选区，包含品种、规格、品牌、产地等属性筛选条件，支持展开收起。
- **排序区**：显示当前筛选条件下的商品总数，排序方式支持按“综合排序”、“价格（由低到高/由高到低）”、“销量”、“发布时间”、“距离”等维度切换。
- **列表展示区**：商品以网格或列表形式展示，每种商品卡片包含商品主图（支持鼠标悬停切换副图）、商品名称、价格信息（起订量、单价、单位）、产地、销量、发布时间、卖家信息、收藏按钮、立即购买/询价按钮等元素。
- **分页区**：支持数字分页和加载更多两种模式，显示总页数和当前页码信息。

**交互功能**：

- 筛选联动：用户选择筛选条件后，列表实时更新，无需手动触发刷新。
- 排序切换：点击排序选项后，列表重新排序并刷新。
- 图片懒加载：商品图片采用懒加载技术，进入可视区域后才加载，减少首屏加载时间。
- 收藏功能：点击收藏按钮，可将商品添加至个人收藏夹；若未登录则跳转至登录页。
- 批量询价：支持勾选多个商品后批量发起询价。

#### 2.2.2 供应信息详情页

供应信息详情页是用户了解商品详情、做出购买决策的核心页面，需提供全面、真实、准确的商品信息。

**页面布局**：

- **面包屑导航**：展示“首页 > 供应 > [一级分类] > [二级分类] > [商品名称]”的路径导航，支持点击返回上级页面。
- **商品基本信息区**：
  - 商品轮播图：展示商品多角度高清图片，支持放大查看、左右滑动切换、图片下载；支持视频展示（若上传了商品视频）。
  - 商品标题：包含商品名称、品种、规格等关键信息；若有活动标识（新品、热销、促销等）在标题旁展示相应标签。
  - 价格信息：展示单价、起订量、计量单位、是否含税等信息；若存在阶梯价格，展开显示不同数量区间的单价。
  - 库存信息：展示当前库存数量和计量单位；若库存紧张，显示“库存紧张”提示。
  - 产地信息：展示商品产地省份和城市，支持点击查看该产地的其他商品。
  - 发布时间：显示信息发布时间和更新时间。
- **店铺信息区**：
  - 店铺头像、店铺名称、店铺类型（个人店铺/企业店铺）、主营类目。
  - 信用等级：采用星级评分或积分制，展示店铺综合评分和各项指标评分。
  - 联系方式：显示在线客服、联系电话（可拨打）、微信号（可复制）等联系方式。
  - 关注店铺按钮：点击后可关注该店铺，关注后在个人中心“我的关注”中查看。
- **商品详情区**：
  - 商品参数表：以表格形式展示商品属性信息，包括品种、规格、等级、保质期、储存条件、种植/养殖方式等。
  - 商品描述：支持富文本编辑，展示商品的详细描述、图片、资质证书等。
  - 包装信息：展示商品包装规格、包装材质、包装图片等。
  - 物流信息：展示卖家可提供的物流方式、发货地到各地区的大致运费参考。
- **交易保障区**：
  - 展示平台提供的交易保障服务图标和说明，如“七天无理由退货”、“质量问题包赔”、“资金托管”等。
  - 点击各保障项可查看详细的服务说明和规则。
- **相似商品推荐区**：
  - 根据商品分类、价格区间、属性特征等维度推荐相似商品，以横向滑动卡片形式展示。
  - 推荐商品数量控制在6-12个，避免过度推荐。
- **看了又看推荐区**：
  - 根据用户浏览历史推荐相关商品，采用商品列表形式展示。
- **底部操作栏**（页面固定定位）：
  - 包含收藏按钮、一键拨号按钮、立即购买按钮、加入购物车按钮、询价按钮。
  - 悬浮于页面底部，在用户滚动页面时始终可见。

**交互功能**：

- 图片放大：点击商品图片弹出全屏图片查看器，支持放大、缩小、拖动查看。
- 分享功能：支持生成商品链接、二维码图片，支持分享至微信、微博、QQ等平台。
- 投诉举报：用户可对商品信息进行举报，平台将根据举报内容进行核实处理。
- 价格议价：采购商可与供应商在线议价，支持发送议价消息和报价单。
- 收藏店铺：点击收藏按钮收藏该店铺，后续可在个人中心查看。

#### 2.2.3 发布供应信息

发布供应信息是供应商用户的核心功能，需提供简洁、高效的发布流程。

**发布流程**：

- **第一步：选择分类**：用户需先选择商品所属的一级分类和二级分类，分类选择将影响后续的属性字段展示。
- **第二步：填写信息**：
  - 商品标题：系统提供标题生成辅助工具，可根据选择的分类和填写的信息自动生成规范标题。
  - 商品图片：上传商品实物图片，要求至少上传一张主图，最多上传9张，支持拖拽排序，图片大小不超过5MB，支持jpg、png格式。
  - 商品视频（可选）：上传商品展示视频，时长不超过60秒，大小不超过100MB。
  - 价格信息：填写单价、计量单位、起订量；支持设置阶梯价格。
  - 库存信息：填写库存数量或选择“无限制”。
  - 产地信息：选择省、市、区县。
  - 商品属性：根据选择的分类，展示相应的属性字段供用户填写，包括品种、规格、等级、产地等。
  - 商品详情：支持富文本编辑器，可插入文字、图片、表格等内容，详细描述商品信息。
  - 包装信息：填写包装规格、包装材质等。
- **第三步：设置物流**：选择支持的物流方式，设置运费模板或包邮规则。
- **第四步：设置保障**：选择开通的交易保障服务，如“七天无理由退货”等。
- **第五步：确认发布**：预览发布的完整信息，确认无误后提交发布。

**校验规则**：

- 必填项校验：标题、图片、价格、单位、起订量、产地为必填项，未填写或填写格式错误时提示用户补充。
- 内容审核：发布的信息需经过平台审核，审核通过后方可展示；审核不通过时需告知具体原因，用户修改后可重新提交。
- 敏感词过滤：系统自动过滤涉黄、涉暴、涉政等敏感内容。
- 重复信息检测：检测与平台上已有信息的重复度，过高重复度的信息将被限制发布。

### 2.3 采购模块

#### 2.3.1 采购需求列表页

采购需求列表页面向供应商展示采购商发布的求购信息，是供应商获取商机的重要渠道。

**页面布局**：

- **顶部搜索区**：与供应列表页类似，包含关键词搜索框、分类选择器、价格区间、地区范围、发布时间范围等筛选条件。
- **采购信息卡片**：以列表形式展示每条采购需求，每条卡片包含：
  - 采购标题：包含采购商品名称、需求量等关键信息。
  - 价格范围：显示采购商的心理价位区间或期望价格。
  - 需求量：显示采购数量和计量单位。
  - 采购区域：显示期望的货源地区。
  - 截止日期：显示采购需求的截止时间。
  - 发布信息：显示采购商头像、名称、认证状态、发布时间。
  - 操作按钮：包含“查看详情”、“立即报价”、“收藏”按钮。
- **分类选项卡**：在页面顶部设置行业分类选项卡，可快速筛选不同行业的采购需求。
- **紧急采购标识**：对发布时间较近或标注为紧急的采购需求，在卡片上显示特殊标识，吸引供应商关注。

#### 2.3.2 采购需求详情页

采购需求详情页供供应商了解采购商的具体需求并决定是否报价。

**页面布局**：

- **面包屑导航**：展示“首页 > 采购 > [一级分类] > [二级分类] > [采购标题]”的路径导航。
- **采购基本信息区**：
  - 采购标题：包含商品名称、品种、需求量等核心信息。
  - 紧急程度：显示采购的紧急程度标识，如“紧急”、“一般”、“长期需求”等。
  - 价格信息：显示期望单价或总价范围、是否含税、是否包物流等。
  - 需求量：显示采购数量、计量单位、最小起订量。
  - 采购区域：显示期望货源地或指定交货地。
  - 发布时间和截止时间：显示采购需求的发布时间和报价截止日期。
  - 期望交货时间：显示采购商期望的交货时间要求。
- **质量要求区**：
  - 详细描述对商品的品质要求、规格要求、检验标准等。
  - 若有样品要求，在此说明。
- **付款信息区**：
  - 付款方式：显示采购商接受的付款方式，如“线上支付”、“货到付款”、“分批付款”等。
  - 付款周期：显示付款时间节点，如“先款后货”、“先货后款”等。
- **采购商信息区**：
  - 采购商头像、名称、企业信息（若企业采购）。
  - 信用评分、认证状态、历史采购记录。
  - 联系方式：显示在线客服、电话等联系方式（部分信息可能需登录后查看）。
- **报价记录区**（若设置为公开）：
  - 显示已收到的报价数量和报价概况。
  - 可查看部分脱敏的报价信息。
- **相似采购推荐区**：推荐其他相似或相关的采购需求。

**交互功能**：

- 立即报价：点击后进入报价填写页面，需填写报价金额、可供货数量、交货时间等信息。
- 收藏采购需求：收藏后可在个人中心查看。
- 分享采购需求：生成分享链接或二维码。
- 举报采购需求：若发现虚假或违规采购需求，可进行举报。

#### 2.3.3 发布采购需求

供应商用户可发布采购需求，吸引供应商主动报价。

**发布流程**：

- **第一步：选择分类**：选择采购商品的分类。
- **第二步：填写需求**：
  - 采购标题：简要描述采购需求。
  - 商品要求：详细描述所需商品的品种、规格、质量要求等。
  - 采购数量：填写需求量、最小起订量。
  - 期望价格：填写单价或总价范围。
  - 产地要求：选择期望的货源地区。
  - 交货地点：填写期望的交货地点。
  - 截止时间：设置报价截止日期。
- **第三步：设置联系方式**：
  - 联系方式：填写联系电话或其他联系方式。
  - 隐私设置：设置联系方式的展示范围（仅平台可见、登录用户可见、公开等）。
- **第四步：确认发布**：预览采购需求信息，确认无误后提交发布。

### 2.4 交易模块

#### 2.4.1 购物车

购物车是用户管理待购买商品、批量结算的核心功能。

**页面布局**：

- **购物车列表区**：
  - 商品以卡片形式展示，包含商品图片、名称、单价、规格、卖家信息、数量选择器（小计器和手动输入）、小计金额、删除按钮等。
  - 支持店铺分组展示，同一店铺的商品归为一组，便于用户管理。
  - 商品列表前设置全选复选框，支持全选当前页面和跨页全选。
- **价格汇总区**（页面右侧或底部固定）：
  - 商品总数量。
  - 商品总价（不含运费）。
  - 运费合计（预估）。
  - 优惠抵扣（优惠券、积分等）。
  - 实付金额。
  - 结算按钮。
- **优惠券使用区**：
  - 显示可使用的优惠券数量，点击展开选择可用优惠券。
  - 系统自动匹配最优优惠券，或用户手动选择。
- **去凑单区**：
  - 若购物车内商品未达到满减条件，显示凑单入口，引导用户添加更多商品。

**交互功能**：

- 数量修改：支持点击加减按钮或手动输入数量修改购买数量，需校验起订量和库存。
- 删除商品：支持单个删除和批量删除，删除前需二次确认。
- 移入收藏：可将商品从购物车移入收藏夹。
- 选择配送方式：不同商品可能支持不同配送方式，可在此选择。
- 更新价格：修改数量或使用优惠券后，实时更新价格汇总。

#### 2.4.2 确认订单页

确认订单页是用户在下单前最后一次核对和补充信息的页面。

**页面布局**：

- **收货地址区**：
  - 展示默认收货地址，包含收货人姓名、电话、详细地址。
  - 若无可用地址，需新增收货地址；若已有多个地址，可切换选择。
  - 支持新增、编辑、删除收货地址。
  - 地址信息包含收货人姓名、手机号码、固定电话（可选）、所在地区（省市区三级联动选择）、详细地址、设为默认地址选项。
- **配送方式区**：
  - 显示可选择的配送方式，包括快递、物流、自提等。
  - 每种配送方式显示运费金额和预计送达时间。
  - 部分商品支持货到付款，在此区域显示选项。
- **商品清单区**：
  - 展示本次购买的商品列表，包含商品图片、名称、规格、单价、购买数量、小计金额。
  - 若存在多店铺商品，按店铺分组展示。
  - 可在此修改购买数量（需校验）。
- **发票信息区**：
  - 选择发票类型（不开发票、增值税普通发票、增值税专用发票）。
  - 填写发票抬头、纳税人识别号、开户银行、账号、注册地址、电话等信息（根据发票类型显示不同字段）。
- **优惠券区**：
  - 选择使用的优惠券，或使用积分抵扣。
  - 显示已节省金额。
- **订单备注区**：
  - 用户可填写订单备注，如特殊配送要求、商品偏好等。
  - 字数限制200字。
- **价格汇总区**：
  - 商品总价。
  - 运费。
  - 优惠券抵扣。
  - 积分抵扣。
  - 其他优惠。
  - 订单总额（大写金额和小写金额）。
- **提交订单按钮**：
  - 显著位置的“提交订单”按钮，点击后进入支付流程。

#### 2.4.3 订单列表页

订单列表页是用户管理所有订单的入口。

**页面布局**：

- **订单状态选项卡**：
  - 全部订单、待付款、待发货、待收货、已完成、已取消、退款/售后等状态选项卡。
  - 每个选项卡显示对应状态的订单数量。
- **订单筛选区**：
  - 时间范围筛选：近一个月、近三个月、近半年、自定义时间。
  - 关键词搜索：支持通过订单号、商品名称、卖家名称搜索订单。
- **订单卡片**：
  - **订单头部**：订单编号、订单时间、订单状态、店铺名称。
  - **商品信息**：展示订单中的商品列表，包含商品图片、名称、规格、单价、数量、小计金额。
  - **订单金额**：显示订单总金额（含运费）和实付金额。
  - **操作按钮**：根据订单状态显示不同操作按钮。
- **订单分页**：支持数字分页和加载更多模式。

**订单状态说明**：

| 订单状态 | 说明 | 可执行操作 |
|---------|------|----------|
| 待付款 | 订单已创建，等待买家付款 | 取消订单、立即付款 |
| 待发货 | 买家已付款，等待卖家发货 | 提醒发货、申请退款 |
| 待收货 | 卖家已发货，等待买家确认收货 | 查看物流、确认收货、申请退款 |
| 已完成 | 买家已确认收货或系统自动确认 | 删除订单、申请售后、评价 |
| 已取消 | 订单已取消（买家取消或超时取消） | 删除订单 |
| 退款中 | 买家发起退款，等待处理 | 查看详情、撤销退款 |
| 退款完成 | 退款已处理完成 | 删除订单 |
| 售后中 | 买家发起售后申请，等待处理 | 查看详情、撤销申请 |
| 售后完成 | 售后处理完成 | 删除订单 |

#### 2.4.4 订单详情页

订单详情页展示订单的完整信息，是用户核对订单、处理售后的核心页面。

**页面布局**：

- **订单状态横幅**：
  - 显著展示当前订单状态和状态说明。
  - 根据状态显示倒计时（如等待付款剩余时间、发货倒计时等）。
  - 显示相关操作按钮（如“去付款”、“确认收货”等）。
- **收货人信息区**：
  - 收货人姓名、手机号码、详细收货地址。
  - 若为自提订单，显示自提点地址和联系人信息。
- **物流信息区**：
  - 物流公司名称和运单号（点击可复制）。
  - 物流跟踪信息：展示物流流转节点，包括发货时间、运输中各节点、到达目的城市、派送中、签收等状态。
  - 支持地图展示发货地和收货地之间的路线。
- **商品信息区**：
  - 订单商品列表，包含商品图片、名称、规格、单价、数量、小计金额。
  - 商品点击可查看商品详情页。
- **发票信息区**：
  - 显示发票类型、发票抬头、纳税人识别号等信息。
  - 支持查看和下载电子发票。
- **订单信息区**：
  - 订单编号（可复制）。
  - 订单创建时间。
  - 付款时间（已付款订单显示）。
  - 发货时间（已发货订单显示）。
  - 收货时间（已收货订单显示）。
- **价格明细区**：
  - 商品总价。
  - 运费。
  - 优惠抵扣（优惠券、积分等）。
  - 订单总金额。
  - 实付金额（显示大写和小写）。
- **买家留言区**：
  - 显示买家在订单备注中填写的信息。
- **卖家备注区**（如有）：
  - 显示卖家添加的订单备注信息。
- **售后服务区**：
  - 申请退款/退货：订单未发货或发货后一定时间内可申请。
  - 申请售后：订单完成后可申请售后维权。
  - 延长收货：物流异常时可申请延长收货时间。

**交互功能**：

- 复制订单信息：订单编号、物流单号等信息支持一键复制。
- 联系卖家：点击可进入与卖家的聊天页面或拨打电话。
- 催发货：订单付款后一定时间内未发货，用户可点击催发货。
- 确认收货：收到货物后点击确认收货。
- 评价：订单完成后可对商品和卖家进行评价。
- 申请发票：部分订单支持申请补开发票。

### 2.5 店铺模块

#### 2.5.1 供应商店铺首页

供应商店铺首页是展示供应商形象和商品的窗口。

**页面布局**：

- **店铺头部区**：
  - 店铺横幅图：支持自定义上传展示店铺形象。
  - 店铺头像、店铺名称、店铺类型标识。
  - 认证状态标识（个人认证、企业认证、优选认证等）。
  - 信用评分：采用星级评分，1-5星显示。
  - 关注店铺按钮和粉丝数量。
  - 店铺简介。
- **联系方式区**：
  - 在线客服入口。
  - 联系电话（可拨打）。
  - 微信号/二维码。
  - 微信小程序码（如有）。
  - 公司地址。
- **动态评分区**：
  - 展示店铺的动态评分数据，包括：
    - 描述相符：与商品描述的相符程度。
    - 发货速度：平均发货时间。
    - 物流服务：物流公司的服务评价。
    - 服务态度：客服服务质量。
  - 同时显示与行业平均水平的对比（如“高于同行XX%”）。
- **在售商品区**：
  - 店铺经营的商品列表展示。
  - 支持按分类、销量、价格排序筛选。
  - 商品卡片包含图片、名称、价格、销量、收藏数等信息。
- **商品分类区**：
  - 店铺自定义的商品分类导航。
  - 点击分类可查看该分类下的所有商品。
- **公司信息区**（企业店铺展示）：
  - 公司名称、成立时间、注册资本、经营范围。
  - 资质证书展示。
  - 工厂/基地图片展示。
- **联系方式区**：
  - 详细联系方式和地址。
  - 地图定位展示。

#### 2.5.2 店铺商品管理页（商家后台）

商家后台的店铺管理功能，供供应商管理自己的店铺和商品。

**商品管理功能**：

- **商品列表**：
  - 展示店铺所有商品，支持按分类、上下架状态、销量筛选。
  - 每条记录包含商品图片、名称、价格、库存、销量、上下架状态、操作按钮等。
  - 支持批量上下架、批量删除、批量移动分类。
- **商品编辑**：
  - 点击编辑进入商品编辑页面，可修改商品的各项信息。
  - 修改后需重新审核（若平台对商品有审核流程）。
- **新增商品**：
  - 与发布供应信息流程类似，填写商品信息后发布。
- **库存管理**：
  - 批量修改库存。
  - 库存预警设置：当库存低于设定值时发送预警通知。
- **价格管理**：
  - 批量调整价格（按金额调整或按比例调整）。
  - 促销活动设置。

### 2.6 搜索模块

#### 2.6.1 全局搜索

全局搜索是用户在平台快速找到目标内容的主要入口。

**搜索功能**：

- **搜索框**：
  - 位于页面顶部醒目位置。
  - 支持关键词输入，实时显示搜索建议。
  - 搜索建议包含历史搜索记录和热门搜索词。
  - 支持搜索类型切换：供应、采购、店铺、资讯等。
- **搜索结果页**：
  - 搜索结果分类展示：供应信息、采购信息、店铺、资讯等标签页切换。
  - 供应/采购信息以商品列表形式展示，支持筛选和排序。
  - 店铺结果展示店铺基本信息。
  - 资讯结果展示文章标题、摘要和发布时间。
- **搜索词高亮**：在搜索结果中突出显示匹配的关键词。
- **搜索发现**：
  - 搜索无结果时，展示热门搜索词和相关分类推荐。
  - 引导用户发现其他内容。

#### 2.6.2 分类浏览

分类浏览为用户提供了按类目结构浏览商品的入口。

**分类体系**：

- **一级分类**：水果、蔬菜、粮油米面、肉类、禽畜、水产、苗木花卉、农资农机、食品加工、农产品加工等。
- **二级分类**：每个一级分类下细分多个二级分类，如水果下设苹果、梨、橙子、葡萄等。
- **三级分类**（部分类目）：更细分的品种或规格分类。

**分类导航组件**：

- **PC端分类导航**：
  - 左侧固定分类树，显示一二级分类层级。
  - 鼠标悬停一级分类时展开二级分类列表。
  - 选中分类后跳转到该分类的商品列表页。
- **移动端分类导航**：
  - 顶部一级分类横向滚动展示。
  - 选中后下方展示二级分类列表。
  - 支持点击展开查看更多二级分类。

### 2.7 行情资讯模块

#### 2.7.1 行情资讯列表页

行情资讯是平台为用户提供农业市场信息服务的核心板块。

**页面布局**：

- **资讯分类选项卡**：
  - 行情动态、市场分析、价格走势、行业报告、政策解读等分类。
  - 每个分类显示当前文章数量。
- **资讯列表**：
  - 文章卡片展示：包含文章封面图、标题、摘要、发布时间、阅读量、点赞数等。
  - 支持大图模式和小图模式切换。
- **热门标签**：
  - 展示当前热门的话题标签，点击可查看相关资讯。
- **推荐阅读**：
  - 根据用户浏览历史推荐相关或热门文章。

#### 2.7.2 行情资讯详情页

资讯详情页展示完整的文章内容。

**页面布局**：

- **文章标题区**：
  - 文章大标题。
  - 文章标签（如“市场分析”、“价格走势”等）。
  - 发布时间和阅读量。
  - 作者信息（如为原创）。
- **文章正文区**：
  - 文章正文内容，支持文字、图片、表格、视频等多种格式。
  - 关键数据图表（如价格走势图、供需对比图等）以可视化形式展示。
  - 引用来源标注。
- **相关阅读区**：
  - 推荐与本文相关的其他文章。

## 五、接口需求规格

### 5.1 接口设计原则

- **RESTful风格**：接口设计遵循RESTful规范，使用HTTP标准方法（GET/POST/PUT/DELETE）。
- **统一响应格式**：所有接口采用统一的JSON响应格式。
- **分页规范**：列表接口支持分页查询，默认每页20条记录。
- **版本控制**：接口URL包含版本号，如/api/v1/goods。
- **安全认证**：敏感接口需携带用户身份Token进行认证。

### 5.2 统一响应格式

```json
{
  "code": 200,
  "message": "success",
  "data": {
    // 业务数据
  },
  "page": {
    "total": 100,
    "page_no": 1,
    "page_size": 20,
    "total_pages": 5
  },
  "timestamp": 1709366400000
}
```

| 接口名称 | 请求方法 | 接口路径 | 说明 |
|---------|---------|---------|------|
| 用户注册 | POST | /api/v1/user/register | 用户注册 |

| 商品列表 | GET | /api/v1/goods/list | 获取商品列表 |
| 商品详情 | GET | /api/v1/goods/{id} | 获取商品详情 |
| 发布商品 | POST | /api/v1/goods | 发布商品 |

| 创建订单 | POST | /api/v1/order | 创建订单 |
